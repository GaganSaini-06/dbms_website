IMPLICIT CURSOR

Definition: An Implicit Cursor is automatically created by Oracle whenever a DML statement (INSERT, UPDATE, DELETE) or a SELECT statement that returns only one row is executed. These cursors are managed internally by PL/SQL and don't require explicit declaration or control.

Cursor Attributes:

  • %FOUND – Returns TRUE if the statement affects at least one row.
  • %NOTFOUND – Returns TRUE if no rows were affected.
  • %ROWCOUNT – Number of rows affected.
  • %ISOPEN – Always FALSE for implicit cursors (they close automatically).

Example:

DECLARE
  v_count NUMBER;
BEGIN
  UPDATE students SET marks = 80 WHERE course = 'BCA';
  v_count := SQL%ROWCOUNT;
  DBMS_OUTPUT.PUT_LINE('Rows Updated: ' || v_count);
END;

Output:

Rows Updated: 3
