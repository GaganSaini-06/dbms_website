FOREIGN KEY

Definition: A FOREIGN KEY is used to link two tables together.
It is a field (or group of fields) in one table that refers to the PRIMARY KEY in another table.
It ensures referential integrity — meaning values in the foreign key must exist in the referenced primary key.

Syntax:

CREATE TABLE child_table ( column_name datatype, FOREIGN KEY (column_name) REFERENCES parent_table(primary_key_column) );

Example:

First, create the parent table:
CREATE TABLE department ( dept_id INT PRIMARY KEY, dept_name VARCHAR(50) );

Now, create the child table with a foreign key:
CREATE TABLE student ( roll_no INT PRIMARY KEY, name VARCHAR(50), dept_id INT, FOREIGN KEY (dept_id) REFERENCES department(dept_id) );
