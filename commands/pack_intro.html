PACKAGES

Definition: A package in PL/SQL is a collection of related procedures, functions, variables, cursors, and other PL/SQL constructs grouped together in a single unit. It consists of two parts: the package specification and the package body.

Advantages of Packages:

  • Modularity: Related code is grouped together.
  • Reusability: Can be used across different programs.
  • Performance: Loaded into memory once and used many times.
  • Encapsulation: Hides implementation details from users.

Syntax: (Specification)

CREATE OR REPLACE PACKAGE package_name IS
  -- Procedure and function declarations
  PROCEDURE hello;
END package_name;

Syntax: (Body)

CREATE OR REPLACE PACKAGE BODY package_name IS
  PROCEDURE hello IS
  BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello from package!');
  END hello;
END package_name;

Example:

-- Create package specification
CREATE OR REPLACE PACKAGE greet_pkg IS
  PROCEDURE greet;
END greet_pkg;

-- Create package body
CREATE OR REPLACE PACKAGE BODY greet_pkg IS
  PROCEDURE greet IS
  BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello Gagan!');
  END greet;
END greet_pkg;

Calling the Procedure from Package:

BEGIN
  greet_pkg.greet;
END;

Output:

Hello Gagan!
