INTRODUCTION TO CURSORS

Definition: A Cursor in PL/SQL is a pointer to the memory area (called context area) where SQL statements are executed. Cursors allow row-by-row processing of query results.

Why Use Cursors?

  • To process multiple rows returned by a SQL query one at a time.
  • To perform operations on each individual row.
  • To have better control over query execution and data handling.

Types of Cursors:

  • 1. Implicit Cursor: Automatically created by PL/SQL when a SQL statement like INSERT, DELETE, UPDATE, or SELECT (single row) is executed.
  • 2. Explicit Cursor: Defined by the user to handle queries that return more than one row. Requires opening, fetching, and closing manually.
  • 3. Parameterized Cursor: A user-defined cursor that accepts parameters to filter data dynamically at runtime. It allows the same cursor to be reused with different input values.

Cursor Lifecycle:

  1. DECLARE the Cursor
  2. OPEN the Cursor
  3. FETCH data from Cursor
  4. CLOSE the Cursor

Example (Just Declaration):

DECLARE
  CURSOR emp_cursor IS
  SELECT emp_id, emp_name FROM employees;
