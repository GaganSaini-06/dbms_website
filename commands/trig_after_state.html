AFTER STATEMENT LEVEL TRIGGER

Definition: An AFTER STATEMENT LEVEL TRIGGER is executed once after the entire SQL operation (INSERT, UPDATE, or DELETE) has been completed, regardless of the number of rows affected.

Use Case: This type of trigger is useful for actions that need to happen after a DML operation is completed — like logging actions, updating audit tables, or sending notifications.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name  
AFTER INSERT OR UPDATE OR DELETE  
ON table_name  
BEGIN  
  -- Trigger logic here  
END;

Example:

This trigger prints a message after any row is deleted from the employees table:

CREATE OR REPLACE TRIGGER log_after_delete  
AFTER DELETE ON employees  
BEGIN  
  DBMS_OUTPUT.PUT_LINE('Delete operation has been completed');  
END;

Key Points:

  • Fires only once for the whole DML operation, not for each row.
  • Cannot access :NEW or :OLD values.
  • Used for global-level activities after statement execution.
