TRIGGER PARAMETERS

Definition: Trigger parameters are the important elements used in a trigger to define when and how the trigger will execute.

Key Parameters in Triggers:

  • BEFORE / AFTER: Specifies whether the trigger fires before or after the DML operation (INSERT, UPDATE, DELETE).
  • INSERT / UPDATE / DELETE: Specifies the DML operation that causes the trigger to fire.
  • ON table_name: Specifies the table the trigger is associated with.
  • FOR EACH ROW / STATEMENT: - ROW level: Executes once for each row affected.
    - STATEMENT level: Executes once for the entire DML operation.
  • :OLD and :NEW: These are used in row-level triggers:
    • :OLD – Refers to the old value before DML operation.
    • :NEW – Refers to the new value after DML operation.

Example:

CREATE OR REPLACE TRIGGER emp_before_update  
BEFORE UPDATE ON employees  
FOR EACH ROW  
BEGIN  
  DBMS_OUTPUT.PUT_LINE('Old Salary: ' || :OLD.salary);  
  DBMS_OUTPUT.PUT_LINE('New Salary: ' || :NEW.salary);  
END;

Explanation: This trigger uses :OLD and :NEW to display salary values before and after the update. The trigger runs before each row is updated.
