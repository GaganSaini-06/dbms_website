INTRODUCTION TO VIEWS

Definition: A View in SQL is a virtual table that is based on the result of a SELECT query. It does not store data physically but displays data stored in tables.

Purpose of Views:

  • To simplify complex queries.
  • To provide a layer of security by restricting access to specific columns or rows.
  • To present a consistent interface even when the underlying table structures change.

Syntax:

CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Example:

CREATE VIEW high_salary_employees AS
SELECT name, salary
FROM employees
WHERE salary > 50000;

Accessing Data from a View:

SELECT * FROM high_salary_employees;

Note: Views can be read-only or updatable depending on how they are created.
