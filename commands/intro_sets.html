INTRODUCTION TO SETS

Definition: In SQL, Set Operators are used to combine the results of two or more SELECT queries. These operators help in handling multiple result sets in a meaningful way.

Rules for Using Set Operators:

  • Each SELECT statement must have the same number of columns.
  • Columns must be of compatible data types.
  • The order of columns should be the same in all SELECT statements.

Types of Set Operators:

  • UNION: Combines results of two SELECT statements and removes duplicates.
  • UNION ALL: Combines results of two SELECT statements without removing duplicates.
  • INTERSECT: Returns only the common records from both SELECT statements.
  • MINUS (EXCEPT in some databases): Returns records from the first SELECT that are not present in the second.

Example (UNION):

SELECT city FROM customers
UNION
SELECT city FROM suppliers;

Note: The default behavior of UNION is to eliminate duplicates. Use UNION ALL if you want to keep them.
