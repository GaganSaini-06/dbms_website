BEFORE ROW LEVEL TRIGGER

Definition: A BEFORE ROW LEVEL TRIGGER is a trigger that is executed automatically before each row is inserted, updated, or deleted from a table. It is often used to validate or modify data before the change is made.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name  
BEFORE INSERT OR UPDATE OR DELETE  
ON table_name  
FOR EACH ROW  
BEGIN  
   -- trigger logic here  
END;

Example:

This trigger ensures that the salary of an employee is never entered below 10000 before inserting data into the table.

CREATE OR REPLACE TRIGGER check_salary  
BEFORE INSERT ON employees  
FOR EACH ROW  
BEGIN  
  IF :NEW.salary < 10000 THEN  
    RAISE_APPLICATION_ERROR(-20001, 'Salary must be at least 10000');  
  END IF;  
END;

Explanation:

  • :NEW refers to the new values being inserted.
  • If the salary is less than 10000, the trigger raises an error and prevents the insertion.
  • It works on each row being inserted.
