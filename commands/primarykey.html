PRIMARY KEY

Definition:The PRIMARY KEY is used to uniquely identify each record in a table.
It automatically combines NOT NULL and UNIQUE constraints, so no duplicate or null values are allowed in the primary key column.

Syntax:

For single column:
CREATE TABLE table_name ( column_name datatype PRIMARY KEY );
For composite key (more than one column):
CREATE TABLE table_name ( column1 datatype, column2 datatype, PRIMARY KEY (column1, column2) );

Example:

CREATE TABLE student ( roll_no INT PRIMARY KEY, name VARCHAR(50) );

Output:

mysql> CREATE TABLE temp_table4 (Rollno INT PRIMARY KEY, Name VARCHAR(35), Age INT);
Query OK, 0 rows affected (0.10 sec)

mysql> INSERT INTO temp_table4 VALUES (90, 'GAGAN', 19);
Query OK, 1 row affected (0.01 sec)

mysql> INSERT INTO temp_table4 VALUES (90, 'ANKIT', 19);
ERROR 1062 (23000): Duplicate entry '90' for key 'temp_table4.PRIMARY'

mysql> INSERT INTO temp_table4 VALUES (NULL, 'ANKIT', 19);
ERROR 1048 (23000): Column 'Rollno' cannot be null
