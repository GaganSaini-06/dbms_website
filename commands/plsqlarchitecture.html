PL/SQL ARCHITECTURE

Definition: PL/SQL architecture defines the internal processing of PL/SQL code inside the Oracle database. It explains how SQL and PL/SQL statements are handled and which engine (SQL engine or PL/SQL engine) executes them.

Components of PL/SQL Architecture:

  • PL/SQL Block: The smallest unit of PL/SQL program.
  • PL/SQL Engine: Responsible for processing procedural statements. If a statement is procedural, it is executed by PL/SQL engine.
  • SQL Engine: If a statement is SQL (like SELECT, INSERT), it is passed to SQL engine for execution.
  • Memory Structures: Includes cursor areas and context areas for managing data during execution.
  • Database Buffer Cache: Frequently accessed data is stored here to speed up performance.

Working Flow:

1. User sends PL/SQL block to Oracle Server
2. PL/SQL Engine checks each statement
3. Procedural statements → Executed by PL/SQL engine
4. SQL statements → Forwarded to SQL engine
5. Output is returned back to the user

Example:

DECLARE
 msg VARCHAR2(30) := 'Hello from PL/SQL';
BEGIN
 DBMS_OUTPUT.PUT_LINE(msg);
END;
/

Note: In the above example, DBMS_OUTPUT.PUT_LINE is handled by the PL/SQL engine as it is a procedural statement. If we used SELECT, that would be executed by the SQL engine.
