BEFORE STATEMENT LEVEL TRIGGER

Definition: A BEFORE STATEMENT LEVEL TRIGGER is fired once before the execution of the triggering SQL statement (INSERT, UPDATE, or DELETE), regardless of how many rows are affected.

Use Case: This type of trigger is commonly used for checking business rules, setting system values, or auditing before a DML operation is performed on a table.

Syntax:

CREATE OR REPLACE TRIGGER trigger_name  
BEFORE INSERT OR UPDATE OR DELETE  
ON table_name  
BEGIN  
  -- Trigger logic here  
END;

Example:

This trigger checks a rule before any UPDATE on the employees table is executed.

CREATE OR REPLACE TRIGGER check_before_update  
BEFORE UPDATE ON employees  
BEGIN  
  DBMS_OUTPUT.PUT_LINE('Update operation is about to begin');  
END;

Explanation:

  • Does not access individual rows — no use of :NEW or :OLD.
  • Fires only once per triggering SQL statement.
  • Useful for tasks that must be done once before the action starts.
