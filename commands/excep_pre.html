PREDEFINED EXCEPTIONS

Definition: Predefined exceptions are built-in exceptions in PL/SQL that are automatically raised when certain common error conditions occur during execution.

These exceptions are implicitly declared by Oracle and can be handled in the EXCEPTION section of a PL/SQL block without needing to define them manually.

Common Predefined Exceptions:

  • NO_DATA_FOUND: Raised when a SELECT INTO statement returns no rows.
  • TOO_MANY_ROWS: Raised when a SELECT INTO statement returns more than one row.
  • ZERO_DIVIDE: Raised when a number is divided by zero.
  • INVALID_CURSOR: Raised when operations are attempted on an unopened cursor.
  • CUSTOMER_ALREADY_EXISTS: This is not a predefined exception; just showing that predefined exceptions are already named by Oracle.

Syntax & Example:

DECLARE
  v_num1 NUMBER := 10;
  v_num2 NUMBER := 0;
  v_result NUMBER;
BEGIN
  v_result := v_num1 / v_num2;  -- This will raise ZERO_DIVIDE
  DBMS_OUTPUT.PUT_LINE('Result: ' || v_result);

EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Error: Cannot divide by zero!');
END;

Output:

Error: Cannot divide by zero!
