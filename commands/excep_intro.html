INTRODUCTION TO EXCEPTION HANDLING

Definition: Exception handling in PL/SQL is a mechanism to handle runtime errors. If an error occurs during program execution, control is passed to the exception-handling section of the block. It helps to keep the program running smoothly without abrupt termination.

Why Exception Handling?

  • To avoid abnormal termination of program.
  • To handle predictable and unpredictable errors.
  • To maintain program flow even after errors.

Structure of Exception Handling in PL/SQL:

DECLARE
   -- Variable declaration
BEGIN
   -- Executable statements
EXCEPTION
   WHEN exception_name THEN
      -- Exception handling statements
END;

Types of Exceptions:

  • Predefined Exceptions: Built-in exceptions in PL/SQL like NO_DATA_FOUND, ZERO_DIVIDE.
  • User-defined Exceptions: Declared explicitly by the programmer using the EXCEPTION keyword.

Example:

DECLARE
   num1 NUMBER := 10;
   num2 NUMBER := 0;
   result NUMBER;
BEGIN
   result := num1 / num2;
   DBMS_OUTPUT.PUT_LINE('Result: ' || result);
EXCEPTION
   WHEN ZERO_DIVIDE THEN
      DBMS_OUTPUT.PUT_LINE('Cannot divide by zero!');
END;

Output:

Cannot divide by zero!
